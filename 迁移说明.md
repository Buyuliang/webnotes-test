# 从 notes-data.json 迁移到目录结构

## 变更说明

应用已从使用单个 `notes-data.json` 文件存储所有笔记，迁移到使用 `notes/` 目录结构，每个笔记存储为独立的 Markdown 文件。

## 主要变更

### 之前（旧版本）
- 所有笔记存储在 `notes-data.json` 文件中
- 文件格式为 JSON 数组
- 每次保存需要更新整个文件

### 现在（新版本）
- 每个笔记存储为 `notes/` 目录下的独立 `.md` 文件
- 支持目录结构（可以创建子文件夹）
- 支持 Markdown 格式
- 每个文件独立管理，减少冲突

## 数据迁移

### 自动迁移
应用会自动检测并使用新的目录结构。如果 `notes/` 目录不存在，会自动创建。

### 手动迁移（如果需要）
如果你有旧的 `notes-data.json` 文件，可以：

1. 打开应用
2. 配置 GitHub Token 和仓库
3. 应用会自动加载目录结构
4. 旧的 `notes-data.json` 文件可以保留（不会被删除，但不再使用）

## 兼容性

- 本地存储（`localStorage`）仍然保留，用于快速访问
- 旧的 `notes-data.json` 文件不会被自动删除，但不再使用
- 如果需要，可以手动删除 `notes-data.json` 文件

## 新功能

1. **目录结构**：可以在 `notes/` 目录下创建子文件夹
2. **独立文件**：每个笔记是独立的文件，减少冲突
3. **Markdown 支持**：笔记文件使用 Markdown 格式
4. **更好的组织**：可以按文件夹组织笔记

## 注意事项

- 旧的 `notes-data.json` 文件不会自动迁移到新格式
- 如果需要迁移旧数据，可以手动创建笔记
- 新创建的笔记会直接保存到 `notes/` 目录
